# 화면 흐름 설계서 (Wire Flow)

본 문서는 **'DTProject'** 서비스의 주요 화면 전환 흐름과 클라이언트 간 상호작용을 정의합니다. 

## 1. 전체 화면 흐름도

```mermaid
graph TD
    Entry((사용자 진입)) --> Home["메인 홈 (아이좋아 스타일)"]
    Home --> Signup["회원가입 (학교코드 입력)"]
    Signup --> Login["로그인"]
    Login --> HomeLoggedIn["메인 홈 (로그인 상태)"]
    
    HomeLoggedIn --> |'디지털 교과서' 버튼 클릭| DT_Home["디지털 교과서 대시보드"]
    
    subgraph "디지털 교과서 (교사)"
        DT_Home --> T_Dashboard["수업 목록 및 리포트 조회"]
        T_Dashboard --> |수업 선택| T_TextbookView["교과서 뷰어 진입"]
        T_TextbookView --> |상단 [수업하기] 버튼 클릭| Start((수업 시작 & 실시간 제어))
    end
    
    subgraph "디지털 교과서 (학생)"
        DT_Home --> S_Bookshelf["내 교과서 목록"]
        S_Bookshelf --> |교재 선택| S_Viewer["교과서 뷰어 진입"]
        Start -.-> |WebSocket 동기화| S_Viewer
        S_Viewer --> |교사가 [시작하기] 클릭 시| S_Sync["교차 화면 강제 추적 및 제어 모드"]
    end

    Start --> |상태 전환| ActiveSync["실시간 수업 세션 활성화"]
```

## 2. 세부 화면 설명

| 단계 | 화면명 | 주요 기능 및 특징 |
| :--- | :--- | :--- |
| **01** | **메인 홈 (Landing)** | '아이좋아' 홈페이지 클론 형태. 상단 GNB(Global Navigation Bar)에 [디지털 교과서] 버튼 배치. |
| **02** | **회원가입/로그인** | 학교 코드를 통한 식별 및 교사/학생 권한 자동 부여. |
| **03** | **교사 대시보드** | 학급 학생 목록, 최근 학습 리포트(AI), 수업 시작/종료 제어. |
| **04** | **학생 교과서 뷰어** | PDF/e-Pub 기반 콘텐츠 뷰어. 우측 사이드바에 필기 도구 및 질문 버튼. |
| **05** | **실시간 동기화 모드** | 교사가 '동기화 시작' 시, 활성화된 학생들의 화면이 교사와 동일한 페이지로 고정 및 판서 공유. |

## 3. UI/UX 연출 전략

-   **저학년 (1-3학년):** 파스텔톤 컬러, 큰 버튼, 귀여운 마스코트 캐릭터 기반 가이드. 텍스트 최소화.
-   **고학년 (4-6학년):** 고채도 강조색, 도구 중심의 툴바(펜, 형광펜, 메모), 자기주도적 위젯 배치.
-   **교사 전용:** 데이터 시각화(차트) 및 실시간 학생 상태 알림(멈춰있는 학생, 진도가 빠른 학생 표시).

---
> [!IMPORTANT]
> 테스트 단계에서는 웹 환경에서 [디지털 교과서] 버튼 하나로 진입하는 심리스(Seamless)한 경험을 우선 구현합니다.
